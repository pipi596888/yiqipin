# yiqipin 后台管理系统 - UI 设计文档

## 概述

本文档描述 yiqipin 拼拼后台管理系统（MVP 版本）的界面设计规范。该系统使用 Vue3 + Ant Design Vue 技术栈，面向商家管理员使用。

---

## 1. 设计原则

### 1.1 核心原则
- **简洁高效**：减少操作步骤，快速达成目标
- **信息清晰**：重要信息突出显示，层次分明
- **一致性**：统一视觉风格和交互模式
- **可用性**：符合用户习惯，降低学习成本

### 1.2 布局原则
- 左重右轻：侧边栏在左侧，功能区在右侧
- 上重下轻：顶部突出关键操作，列表展示在下方
- 卡片化：相关内容使用卡片容器分组

### 5.1 登录页面

#### 布局结构
```
┌─────────────────────────────────────────┐
│                                         │
│            ┌───────────────┐            │
│            │    Logo       │            │
│            │   系统名称    │            │
│            ├───────────────┤            │
│            │  用户名输入   │            │
│            │  密码输入     │            │
│            │  验证码输入   │            │
│            │  [登录按钮]   │            │
│            │  记住我 □     │            │
│            └───────────────┘            │
│                                         │
└─────────────────────────────────────────┘
```

#### 设计要点
1. **整体风格**：简洁大气、垂直居中
2. **视觉焦点**：登录按钮使用主色，视觉突出
3. **输入验证**：输入时即时校验，错误信息显示在输入框下方
4. **交互反馈**：登录按钮点击后显示 loading 状态

#### Ant Design Vue 组件建议
- `a-form`：表单容器，配置 layout="vertical"
- `a-input`：用户名、密码输入框，带前缀图标
- `a-checkbox`：记住我选项
- `a-button`：登录按钮，type="primary"
- `a-message`：登录错误提示

#### 组件使用示例
```vue
<a-form :model="formState" layout="vertical" @finish="handleLogin">
  <a-form-item name="username" :rules="[{ required: true, message: '请输入用户名' }]">
    <a-input v-model:value="formState.username" placeholder="用户名" size="large">
      <template #prefix><UserOutlined /></template>
    </a-input>
  </a-form-item>
  <a-form-item name="password" :rules="[{ required: true, message: '请输入密码' }]">
    <a-input-password v-model:value="formState.password" placeholder="密码" size="large">
      <template #prefix><LockOutlined /></template>
    </a-input-password>
  </a-form-item>
  <a-form-item>
    <a-checkbox v-model:checked="formState.remember">记住我</a-checkbox>
  </a-form-item>
  <a-form-item>
    <a-button type="primary" html-type="submit" size="large" block :loading="loading">
      登录
    </a-button>
  </a-form-item>
</a-form>
```

#### 设计要点
1. **列表展示**：表格形式，图片列显示商品缩略图

3. **批量操作**：支持批量上下架、批量删除

#### 表格列配置
| 列名 | 宽度 | 内容 |
|------|------|------|
| 选择框 | 50px | 复选框 |
| 商品信息 | 300px | 图片(60x60) + 名称 + SKU |
| 分类 | 120px | 分类名称 |
| 价格 | 100px | 销售价(红色加粗) |
| 库存 | 80px | 库存数量，库存不足标红 |
| 状态 | 80px | 状态标签 |
| 操作 | 150px | 编辑、删除、上下架 |

#### Ant Design Vue 组件建议
- `a-table`：数据表格，使用 columns、dataSource、pagination
- `a-input`：搜索输入框，带搜索按钮
- `a-select`：筛选下拉框
- `a-button`：操作按钮
- `a-tag`：状态标签
- `a-popconfirm`：删除确认弹窗
- `a-space`：按钮间距

#### 组件使用示例
```vue
<a-table
  :columns="columns"
  :data-source="products"
  :row-selection="rowSelection"
  :pagination="pagination"
  row-key="id"
>
  <template #bodyCell="{ column, record }">
    <template v-if="column.key === 'image'">
      <img :src="record.image" style="width: 60px; height: 60px; object-fit: cover" />
    </template>
    <template v-if="column.key === 'price'">
      <span style="color: #FF4D4F; font-weight: 600">¥{{ record.price }}</span>
    </template>
    <template v-if="column.key === 'status'">
      <a-tag :color="record.status === 1 ? 'green' : 'red'">
        {{ record.status === 1 ? '在售' : '下架' }}
      </a-tag>
    </template>
    <template v-if="column.key === 'action'">
      <a-space>
        <a @click="handleEdit(record)">编辑</a>
        <a-popconfirm title="确定删除?" @confirm="handleDelete(record.id)">
          <a style="color: #FF4D4F">删除</a>
        </a-popconfirm>
      </a-space>
    </template>
  </template>
</a-table>
```

```
│  共 100 条  第 1/10 页  [<] [>]  每页 [20 ▼] 条    │
└─────────────────────────────────────────────────────┘
```

#### 设计要点
1. **头像展示**：用户头像列显示圆形头像
2. **用户等级**：使用 VIP 标签或图标区分等级
3. **积分显示**：显示用户积分数量
4. **状态管理**：正常/禁用状态

#### 表格列配置
| 列名 | 宽度 | 内容 |
|------|------|------|
| 选择框 | 50px | 复选框 |
| 用户信息 | 200px | 头像(40x40) + 昵称 + 用户ID |
| 昵称 | 120px | 用户昵称 |
| 手机 | 120px | 手机号（脱敏显示） |
| 等级 | 100px | VIP等级标签 |
| 积分 | 100px | 积分数值 |
| 状态 | 80px | 状态标签 |
| 操作 | 120px | 编辑、禁用 |

#### Ant Design Vue 组件建议
- `a-table`：数据表格
- `a-avatar`：用户头像，使用 :size="40"
- `a-tag`：等级标签，不同等级不同颜色
- `a-badge`：积分徽章
- `a-switch`：用户状态开关

#### 组件使用示例
```vue
<a-table :columns="columns" :data-source="users" row-key="id">
  <template #bodyCell="{ column, record }">
    <template v-if="column.key === 'userInfo'">
      <a-space>
        <a-avatar :src="record.avatar" :size="40" />
        <div>
          <div>{{ record.nickname }}</div>
          <div style="color: #8C8C8C; font-size: 12px">ID: {{ record.id }}</div>
        </div>
      </a-space>
    </template>
    <template v-if="column.key === 'level'">
      <a-tag :color="getLevelColor(record.level)">
        <GemOutlined /> {{ record.levelName }}
      </a-tag>
    </template>
    <template v-if="column.key === 'status'">
      <a-switch
        v-model:checked="record.status"
        checked-children="正常"
        un-checked-children="禁用"
        @change="handleStatusChange(record)"
      />
    </template>
  </template>
</a-table>
```

## 6. 组件使用规范

### 6.1 按钮规范

| 类型 | 使用场景 | 样式 |
|------|----------|------|
| 主按钮 | 主要操作：提交、登录、保存 | type="primary"，主色背景 |
| 默认按钮 | 次要操作：取消、返回 | type="default" |
| 链接按钮 | 辅助操作：查看详情、编辑 | type="link" |
| 危险按钮 | 危险操作：删除、禁用 | danger 属性 |

### 6.2 表单规范
- 必填项使用红色星号 `*` 标记
- 错误信息显示在输入框下方
- 表单布局使用 `a-form` 的 `layout="horizontal"`
- 标签位置使用 `labelAlign="right"`

### 6.3 表格规范
- 表头使用深色背景
- 斑马纹使用极浅的灰色
- 操作列固定在右侧
- 翻页组件放置在表格下方右侧

### 6.4 弹窗规范
- 确认弹窗使用 `a-modal`
- 较小的确认使用 `a-popconfirm`
- 表单弹窗宽度建议 600px
- 详情弹窗宽度建议 800px
