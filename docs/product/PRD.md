## 1. 产品概述

### 1.1 产品背景

一期buy（yiqipin）是一个面向消费者的综合电商平台，为用户提供便捷的线上购物体验。系统采用前后端分离架构，前端使用 UniApp 框架实现多端统一开发（支持 H5、小程序、App），后端使用 Spring Boot 3.x 构建 RESTful API 服务。

当前项目已完成基础框架搭建，包括用户端核心页面的 UI 设计和后端 Controller 层的接口定义。本文档旨在明确 MVP 阶段的产品需求，指导开发团队进行功能实现。

### 1.2 产品定位

- **产品名称**: 一期buy（yiqipin）
- **产品类型**: B2C 电商平台
- **核心价值**: 为用户提供便捷、多端统一的购物体验
- **目标市场**: 国内消费者，初期聚焦精品商品

### 1.3 目标用户

| 用户群体 | 特征 | 需求 |
|----------|------|------|
| 普通消费者 | 18-45岁，有线上购物需求 | 浏览商品、下单购买、查看订单 |
| 平台商家 | 需要上架商品、管理库存 | 商品上下架、价格调整、库存管理 |
| 平台运营 | 需要分析销售数据 | 查看报表、运营数据分析 |

### 1.4 MVP 定义

**MVP（最小可行产品）阶段目标**：

1. 用户端完成核心购买流程（注册登录 → 浏览商品 → 加购 → 下单 → 模拟支付）
2. 管理端完成商品基础管理功能
3. 前后端数据打通，关闭 Mock 模式

**MVP 不包含**：
- 真实支付网关集成
- 优惠券/营销活动
- 完整客服系统
- 用户评价系统
- 物流跟踪

## 9. 验收标准

### 9.1 MVP 验收清单

- [ ] 用户可以完成注册登录流程
- [ ] 用户可以浏览商品列表和详情
- [ ] 用户可以将商品加入购物车
- [ ] 用户可以创建订单并完成模拟支付
- [ ] 用户可以管理收货地址
- [ ] 管理员可以登录后台
- [ ] 管理员可以新增、编辑、删除商品
- [ ] 管理员可以上下架商品
- [ ] 管理员可以修改商品库存和价格

### 9.2 测试场景

1. **注册登录流程**: 新用户注册 → 登录 → 退出 → 重新登录
2. **购买流程**: 浏览 → 加购 → 确认订单 → 支付 → 查看订单
3. **地址管理**: 新增地址 → 编辑 → 设为默认 → 删除
4. **商品管理**: 新增商品 → 编辑 → 上架 → 下架 → 删除
